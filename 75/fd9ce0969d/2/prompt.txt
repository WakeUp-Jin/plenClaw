    20|# PineClaw Skill 客户端支持方案
    21|
    22|## 设计理念
    23|
    24|官方文档定义了两种激活方式。我们选择 **File-read activation**（最简单的一种）：
    25|
    26|> The model calls its standard file-read tool with the SKILL.md path from the catalog.
    27|> No special infrastructure needed.
    28|
    29|PineClaw 已有 `ReadFile` 工具。所以 **Python 代码只负责 Tier 1（告诉模型有哪些 Skill）**，Tier 2（读 SKILL.md）和 Tier 3（读 references / 执行 scripts）完全由模型通过现有工具自主完成。
    30|
    31|---
    32|
    33|## 渐进式三层加载
    34|
    35|```mermaid
    36|graph LR
    37|    subgraph tier1 ["Tier 1: Catalog — 代码负责"]
    38|        A["启动时扫描目录"] --> B["解析 frontmatter"]
    39|        B --> C["构建 XML catalog + 行为指令"]
    40|        C --> D["注入 system prompt segment"]
    41|    end
    42|
    43|    subgraph tier2 ["Tier 2: Instructions — 模型自主"]
    44|        E["模型看到 catalog 描述"] --> F["模型判断相关"]
    45|        F --> G["模型调用 ReadFile(SKILL.md 路径)"]
    46|        G --> H["获得完整 Skill 指令"]
    47|    end
    48|
    49|    subgraph tier3 ["Tier 3: Resources — 模型自主"]
    50|        H --> I["SKILL.md 引用 scripts/xxx.sh"]
    51|        I --> J["模型调用 Bash 执行"]
    52|        H --> K["SKILL.md 引用 references/yyy.md"]
    53|        K --> L["模型调用 ReadFile 读取"]
    54|    end
    55|```

    72|## 注入 system prompt 的具体内容
    73|
    74|这是整个方案的核心产物。当扫描到 Skill 后，以下 XML 文本作为一个 `PromptSegment` 注入 system prompt：
    75|
    76|```xml
    77|<agent_skills>
    78|The following skills provide specialized instructions for specific tasks.
    79|When a task matches a skill's description, use the ReadFile tool to load
    80|the SKILL.md at the listed fullPath before proceeding.
    81|When a skill's instructions reference relative paths (like scripts/run.sh
    82|or references/guide.md), resolve them against the skill's directory
    83|(the parent directory of the SKILL.md file) and use absolute paths
    84|in tool calls.
    85|Only load a skill when the current task is relevant. Do NOT load all
    86|skills preemptively.
    87|
    88|<available_skills>
    89|  <skill name="frontend-design" fullPath="/Users/xjk/.claude/skills/frontend-design/SKILL.md">
    90|    Create distinctive, production-grade frontend interfaces with high design quality.
    91|    Use this skill when the user asks to build web components, pages, or applications.
    92|  </skill>
    93|  <skill name="backend-api" fullPath="/Users/xjk/.agents/skills/backend-api/SKILL.md">
    94|    Provides backend API development guidance with scaffolding support.
    95|    Use when the user asks to create API, implement routes, or build backend services.
    96|  </skill>
    97|</available_skills>
    98|</agent_skills>
    99|```
   100|
   101|结构说明：
   102|- `<agent_skills>` 根标签，包含**行为指令**和**技能列表**
   103|- 行为指令告诉模型三件事：(1) 用 ReadFile 加载 SKILL.md，(2) 相对路径要转成绝对路径，(3) 只在相关时加载
   104|- `<available_skills>` 包含所有发现的 Skill
   105|- 每个 `<skill>` 有 `name`（标识）、`fullPath`（模型用来 ReadFile 的路径）、内部文本是 description
   106|- 如果没发现任何 Skill，**不注入这段内容**（避免空标签困惑模型）
   107|
