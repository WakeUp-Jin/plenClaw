    38|# 记忆模块完整实现计划
    39|
    40|## 一、设计理念
    41|
    42|记忆模块是用户独享的宝贵数据，用户拥有完全控制权。整个模块采用 **Skill 规范**存储，方便用户迁移到任何支持 Skill 加载的 Agent 系统。
    43|
    44|个人助手 Agent 没有"会话"概念——陪伴是永久长期的，不是任务驱动的短期交互。因此用"短期记忆"替代传统的"会话记录"。
    45|
    46|### 4 层渐进式披露
    47|
    48|1. **SKILL.md 元信息**：对记忆模块的简单描述，Skill 扫描时读取
    49|2. **SKILL.md 正文**：长期记忆文件索引 + 短期记忆位置说明，Agent 按需加载
    50|3. **长期记忆文件**：4 种类型各一个 .md 文件，来源于对短期记忆的摘要总结，含溯源链接
    51|4. **短期记忆文件**：完整的 .jsonl 对话记录，按天/月组织
    52|
    53|### 代码现状
    54|
    55|- `ShortMemoryStore`：时间戳文件夹 + `history.jsonl`，百分比裁剪压缩
    56|- `LocalMemoryStore`：单个 `memory.md`，`append` / `rewrite` 操作
    57|- `~/.pineclaw/memory/` 下的现有数据可以删除（开发中）
    58|

   143|### 4.2 重写 `ShortTermMemoryContext`
   144|
   145|当前文件：`[src/core/context/modules/short_term_memory.py](src/core/context/modules/short_term_memory.py)`
   146|
   147|**核心变更**：
   148|
   149|- `_load_memory()` 改为基于 Token 预算的分层加载逻辑
   150|- 加载顺序：今天的 jsonl -> 往回逐天（遇到摘要加载摘要跳过日记录）-> 预算用完停止
   151|- 初始加载预算 = 短期记忆总容量 x 70%（预留 30% 给当前对话增长）
   152|- `needs_compression()` 判断条件：已用容量 >= 总容量 x 85%
   153|- `compress()` 改为按周/月/年压缩逻辑（不再是百分比裁剪）
   154|- 去掉 checkpoint 相关逻辑，压缩结果直接写入摘要文件
   155|
   156|### 4.3 压缩策略改造
   157|
   158|当前文件：`[src/core/context/utils/compressor.py](src/core/context/utils/compressor.py)`
   159|
   160|**三层压缩，各自的输入和侧重点不同**：
   161|
   162|- **周摘要**：输入 = 本周日记录原文。侧重"这周发生了什么，时间线是什么"，追求事件完整性
   163|- **月摘要**：输入 = 本月日记录原文 + 已生成的周摘要。侧重"这个月的核心决策和变化"，追求关键节点。用日记录做输入是为了避免周摘要的"稀释"丢失细节
   164|- **年摘要**：输入 = 月摘要。侧重"这一年最重要的里程碑"。从上下文窗口成本考虑，输入月摘要是最优解
   165|
   166|**新增方法**：
   167|
   168|- `compress_to_week_summary(daily_jsonl_paths, summarize_fn) -> str`：读取连续 N 天的日记录，压缩为周摘要 Markdown
   169|- `compress_to_month_summary(daily_jsonl_paths, week_summary_texts, summarize_fn) -> str`：读取本月日记录 + 周摘要，压缩为月摘要
   170|- `compress_to_year_summary(month_summary_texts, summarize_fn) -> str`：读取多个月摘要，压缩为年摘要
   171|
   172|**压缩触发逻辑**（在 `ShortTermMemoryContext.compress()` 中实现）：
   173|
   174|1. 已用容量 >= 85% 触发压缩
   175|2. 检查已加载内容的结构：
   176|  - **存在日记录（不含今天、昨天）** -> 压缩最早连续 7 天日记录为周摘要
   177|  - **除今天和昨天外全是周摘要** -> 触发月压缩，将该月的日记录+周摘要合并为月摘要
   178|  - **最早已是月摘要** -> 将多个月摘要压缩为年摘要
   179|3. 用 LOW 模型 `simple_chat` 做摘要
   180|4. 摘要写入磁盘，从内存移除被压缩的内容，替换为摘要
   181|5. 压缩后检查是否 < 75%，不够则继续下一轮
   182|
   183|### 4.4 CompressionConfig 扩展
   184|
   185|当前文件：`[src/core/context/types.py](src/core/context/types.py)`
   186|
   187|在 `CompressionConfig` 中增加：
   188|
   189|- `initial_load_ratio: float = 0.70`
   190|- `days_per_week_summary: int = 7`
