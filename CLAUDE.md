# 项目规则

## 用户背景

我是 Python 新手，借助这个项目边练边学。

## 核心原则：先解释，再执行

**在修改任何代码之前，必须：**

1. **解释清楚问题**：这个错误/需求是什么意思，背后的原理是什么
2. **说明涉及的文件和代码**：哪个文件的哪一行，它的作用是什么
3. **给出完整方案**：打算怎么改，改动的理由，可能的影响
4. **等待我确认**：我说"可以"或"执行"之后才能动代码

## 解释风格

- 越详细越好，不要假设我懂
- 遇到 Python 概念（装饰器、dataclass、async/await 等）要顺带解释
- 遇到多种方案时，列出对比，说明推荐理由
- 代码改动后说明改了什么、为什么这样改

## Git 操作

- 不要自动操作 git，只有我明确要求时才可以执行 git 命令
