# 存储模块架构与开发指南

## 概述

存储模块是 PineClaw 的数据持久化层，负责所有本地文件的读写。采用**统一基类 + 三个专用 Store**的架构，将 I/O 操作与业务逻辑（上下文模块、Agent）解耦。

设计原则：
- 纯本地文件存储，不依赖外部服务（替代此前的飞书云文档方案）
- 同步 I/O，不引入 async 开销（本地磁盘操作无需异步）
- 每个 Store 职责单一，对外暴露语义化 API，内部封装文件格式细节
- 单用户场景（每个用户独立运行 Docker 实例）

---

## 概念体系

存储模块围绕三个核心概念组织：

| 概念 | 存储类 | 文件格式 | 用途 |
|------|--------|----------|------|
| **Conversation（对话）** | `ConversationStore` | JSONL | 短期记忆 — 当前/历史对话消息 |
| **Memory（记忆）** | `LocalMemoryStore` | Markdown | 长期记忆 — 用户画像、偏好、事实 |
| **Config（配置）** | `ConfigStore` | JSON | 运行时配置 — 模型参数、路径等 |

### 为什么 Conversation 而非 Session

此前短期记忆文件以 `session_` 命名。改为 `conv_`（Conversation）的理由：

1. **用户心智模型**："开始一段新对话"比"新建 session"更直觉
2. **避免歧义**：session 在 Web 领域专指认证会话，容易混淆
3. **领域一致**：与 LLM 行业通用的 conversation 概念对齐

## 模块详解

### 1. IStorage 基类

**文件**：`src/storage/base.py`

所有 Store 的公共父类，提供目录管理和基础文件 I/O 原语。初始化时自动创建目标目录。

```python
class IStorage:
    def __init__(self, base_dir: str | Path) -> None
    def exists(self, filename: str) -> bool
    def read_text(self, filename: str) -> str
    def write_text(self, filename: str, content: str) -> None
    def append_text(self, filename: str, content: str) -> None
    def delete(self, filename: str) -> bool
    def list_files(self, pattern: str = "*") -> list[Path]
```

设计要点：
- `read_text` 在文件不存在时返回空字符串（而非抛异常），调用方不需要判断文件是否存在
- `delete` 使用 `missing_ok=True`，删除不存在的文件不报错
- 所有异常内部 catch 并 log，不向上抛出（避免 I/O 错误中断业务流）

### 2. ConversationStore — 对话存储

**文件**：`src/storage/conversation_store.py`

管理对话历史的核心 Store。每段对话对应一个 `.jsonl` 文件（append-only），支持压缩检查点。

**双重继承**：同时继承 `IStorage`（文件操作）和 `IContextStorage`（上下文接口），使 `ShortTermMemoryContext` 无需修改即可使用。

#### 文件命名规则

```
conv_{YYYYMMDD}_{HHMMSS}.jsonl              # 对话消息
conv_{YYYYMMDD}_{HHMMSS}.checkpoint.json    # 压缩检查点
```

`conv_id` = 时间戳部分（如 `20260321_143022`）

#### IContextStorage 接口实现

| 方法 | 说明 |
|------|------|
| `append(message)` | 追加一条消息到当前对话文件 |
| `load_all()` | 加载当前对话的所有消息 |
| `load_from_line(n)` | 从第 n 行开始加载（配合压缩检查点使用） |
| `count_lines()` | 返回当前对话的消息总数 |
| `save_checkpoint(summary, line)` | 保存压缩检查点（摘要 + 起始行号） |
| `load_checkpoint()` | 加载检查点，不存在返回 None |
| `new_conversation()` | 创建新对话文件，旧文件保留在磁盘 |

#### 查询管理方法

| 方法 | 说明 |
|------|------|
| `list_conversations()` | 返回所有对话的元数据列表（id、路径、创建时间、消息数），按时间倒序 |
| `get_conversation(conv_id)` | 读取指定对话的所有消息 |
| `delete_conversation(conv_id)` | 删除指定对话及其检查点文件 |
| `current_conversation_id` | 当前对话的 id（时间戳字符串） |

#### ConversationMeta 数据结构

```python
@dataclass
class ConversationMeta:
    conv_id: str          # 时间戳 id，如 "20260321_143022"
    path: Path            # 文件路径
    created_at: str       # ISO 格式时间
    message_count: int    # 消息条数
```

#### 初始化行为

启动时查找 `data/conversations/` 下最新的 `.jsonl` 文件继续使用；若目录为空则自动创建新文件。

### 3. LocalMemoryStore — 长期记忆

**文件**：`src/storage/memory_store.py`

管理用户长期记忆，存储在单个 `memory.md` 文件中。内部维护内存缓存，避免重复读盘。

```python
class LocalMemoryStore(IStorage):
    def get_memory_text(self) -> str       # 读缓存（零 I/O）
    def append_memory(content) -> bool     # 追加内容到文件尾部
    def replace_memory(content) -> bool    # 全量覆写
    def clear_memory() -> bool             # 清空
    def reload() -> None                   # 强制从磁盘重载缓存
    @property memory_path -> Path
    @property is_empty -> bool
```

#### 缓存策略

```
初始化：read_text("memory.md") → _cache
读取：直接返回 _cache（零磁盘 I/O）
写入：写文件 + 更新 _cache（保证一致性）
重载：reload() 强制从磁盘读取（外部编辑后使用）
```

#### memory.md 格式约定

```markdown
# PineClaw Memory

## 用户画像
- 名字：小明
- 角色：后端开发工程师

## 工作习惯
- 每天 10 点开始工作

## 重要事实
- [2026-03-21] 正在开发 PineClaw 项目

## 偏好指令
- 回复使用中文
```

- 一级标题固定为 `# PineClaw Memory`
- 二级标题为分类（LLM 可自行扩展新分类）
- 重要事实条目带日期 `[YYYY-MM-DD]`

#### 与旧版飞书 MemoryStore 的区别

| 维度 | 旧版（飞书） | 新版（本地） |
|------|-------------|-------------|
| 存储位置 | 飞书云文档 | 本地 `data/memory/memory.md` |
| I/O 模式 | async（网络请求） | sync（本地文件） |
| 初始化 | 需要 `await initialize()`（创建文件夹/文档） | 构造函数即完成（自动建目录 + 读缓存） |
| API 调用 | 启动 2-4 次，写入 1 次/次 | 0（纯本地） |
| 可读性 | 需登录飞书查看 | 直接打开 `.md` 文件 |

### 4. ConfigStore — 配置存储

**文件**：`src/storage/config_store.py`

对 `config.json` 的读写封装，支持 dot-notation 路径访问。

```python
class ConfigStore:
    def load() -> dict[str, Any]              # 从磁盘加载（带缓存）
    def save(data=None) -> None               # 写回磁盘
    def get(key_path, default=None) -> Any    # 读：如 "llm.models.high.temperature"
    def set(key_path, value) -> None          # 写 + 自动持久化
```

用法示例：

```python
store = ConfigStore("./config.json")
temp = store.get("llm.models.high.temperature", 0.7)
store.set("chat.max_token_estimate", 80000)  # 自动写入磁盘
```

`ConfigStore` 不继承 `IStorage`（配置文件是单文件，不需要目录管理能力）。

## 初始化流程

在 `main.py` 的 `startup()` 中，存储模块的初始化顺序：

```
1. LocalMemoryStore(base_dir=settings.memory_dir)
   ├── 自动创建 data/memory/ 目录
   └── 读取 memory.md 到内存缓存

2. register_memory_tools(tool_manager, memory_store)
   └── 注册 memory 工具（append / rewrite）

3. ConversationStore(base_dir=settings.conversations_dir)
   ├── 自动创建 data/conversations/ 目录
   └── 查找最新 .jsonl 文件 或 创建新文件

4. ShortTermMemoryContext(storage=conversation_storage, ...)
   └── 通过 IContextStorage 接口使用 ConversationStore

5. LongTermMemoryContext(memory_store=memory_store)
   └── 直接使用 LocalMemoryStore
```

## 数据流

### 对话消息流

```
用户消息
  │
  ▼
ContextManager.append_message()
  │
  ▼
ShortTermMemoryContext.append_message()
  │
  ├── 内存：self._items.append(item)
  └── 磁盘：ConversationStore.append(message)
            → conv_20260321_143022.jsonl 追加一行 JSON

LLM 回复
  │
  ▼
同样路径追加到 .jsonl 文件
```

### 压缩流

```
tokens 超阈值
  │
  ▼
ShortTermMemoryContext.compress()
  │
  ├── LLM 生成摘要
  ├── ConversationStore.save_checkpoint(summary, line)
  │   → conv_20260321_143022.checkpoint.json
  └── 内存中只保留最近 N 条消息 + 摘要
```

### 长期记忆流

```
LLM 判断需要记住信息
  │
  ▼
调用 memory 工具 (action=append)
  │
  ▼
memory_tools._handle_append()
  │
  ▼
LocalMemoryStore.append_memory()
  ├── 磁盘：data/memory/memory.md 追加内容
  └── 缓存：self._cache 同步更新

下一轮对话
  │
  ▼
LongTermMemoryContext.format()
  │
  ▼
LocalMemoryStore.get_memory_text()  ← 直接读缓存，零 I/O
```

### 新建对话流

```
用户发送 "清空对话" / "/clear"
  │
  ▼
SimpleAgent._handle_clear()
  │
  ▼
ContextManager.clear_conversation()
  │
  ▼
ShortTermMemoryContext.clear()
  │
  ├── 清空内存 items + summary
  └── ConversationStore.new_conversation()
      ├── 旧文件 conv_20260321_143022.jsonl 保留
      └── 新建 conv_20260321_150000.jsonl
```

## 与上下文模块的关系

存储模块是上下文模块的"持久化后端"。两者通过 `IContextStorage` 接口解耦：

```
┌─────────────────────────┐
│   core/context/         │
│                         │
│   ShortTermMemoryContext │──── IContextStorage 接口 ────┐
│   LongTermMemoryContext  │──── 直接依赖 ────────────────┤
│                         │                              │
└─────────────────────────┘                              │
                                                         ▼
                                              ┌─────────────────────┐
                                              │   storage/          │
                                              │                     │
                                              │   ConversationStore │
                                              │   LocalMemoryStore  │
                                              └─────────────────────┘
```

`IContextStorage`（定义在 `core/context/storage/base.py`）是上下文模块与存储层的契约接口。`ConversationStore` 同时实现 `IStorage` 和 `IContextStorage`，使得替换底层存储实现时上下文模块无需改动。

## memory 工具

LLM Agent 通过统一的 `memory` 工具操作长期记忆，定义在 `core/tool/memory_tools.py`：

| action | 说明 | 必填参数 |
|--------|------|---------|
| `append` | 追加一条记忆 | `section`（分类）、`content`（内容） |
| `rewrite` | 整理重写整个记忆文件 | `content`（完整 Markdown，须以 `# PineClaw Memory` 开头） |

工具内部调用 `LocalMemoryStore` 的同步方法，通过 `async def _async_handler` 包装以满足 `ToolManager` 的 `Awaitable` 类型要求。

## 迁移说明

### 从旧版（飞书 MemoryStore）迁移

1. **对话文件**：`data/chat_history/session_*.jsonl` → `data/conversations/conv_*.jsonl`（文件内容格式不变，仅文件名前缀改变）
2. **长期记忆**：飞书云文档 `memory.md` → 本地 `data/memory/memory.md`（手动导出或重新积累）
3. **配置字段**：`chat.history_dir` → `chat.conversations_dir`，新增 `chat.memory_dir`
4. **代码引用**：`JsonlContextStorage` → `ConversationStore`，飞书 `MemoryStore` → `LocalMemoryStore`
5. **接口变更**：`IContextStorage.new_session()` → `new_conversation()`

旧的飞书 `MemoryStore`（`src/memory/memory_store.py`）保留在代码库中但不再在默认启动流程中使用。
