# 各模块详解

## 1. BaseContext[T] — 泛型基类

所有上下文模块继承自 `BaseContext[T]`，泛型 `T` 表示单条存储项的类型。

**文件**: `src/core/context/base.py`

**标准方法**:

| 方法 | 说明 |
|------|------|
| `add(item)` | 添加一个存储项 |
| `get(index)` | 按索引获取，越界返回 None |
| `get_all()` | 获取所有存储项的拷贝 |
| `clear()` | 清空所有存储项 |
| `remove_last()` | 移除并返回最后一项 |
| `replace(items)` | 用新列表替换所有存储项 |
| `slice(start, end)` | 切片操作 |
| `count()` | 存储项数量 |
| `is_empty()` | 是否为空 |
| `format()` | **抽象方法**，子类实现：将存储项格式化为 `list[ContextItem]` |

**子类对应关系**:

| 子类 | T 类型 | 用途 |
|------|--------|------|
| SystemPromptContext | PromptSegment | 系统提示词片段 |
| LongTermMemoryContext | str | 长期记忆文本 |
| ShortTermMemoryContext | ContextItem | 对话历史消息 |
| ToolContext | ContextItem | 当前轮次工具消息 |

## 2. SystemPromptContext — 分段式系统提示词

**文件**: `src/core/context/modules/system_prompt.py`

系统提示词由多个 `PromptSegment` 组成，每段有独立的 ID、优先级和启用状态。

### PromptSegment 结构

```python
@dataclass
class PromptSegment:
    id: str           # 唯一标识
    content: str      # 提示词内容
    priority: int     # 越大越靠前（默认 5）
    enabled: bool     # 是否启用（默认 True）
```

### 核心段

构造时自动创建 `core` 段（priority=100），包含 PineClaw 的默认系统提示词。

### 管理方法

| 方法 | 说明 |
|------|------|
| `register_segment(segment)` | 注册新段，同 id 的旧段会被替换 |
| `update_segment(id, content)` | 更新指定段的内容 |
| `remove_segment(id)` | 移除指定段 |
| `enable_segment(id)` | 启用指定段 |
| `disable_segment(id)` | 禁用指定段 |
| `get_segment(id)` | 获取指定段 |
| `get_prompt()` | 返回组装后的完整提示词字符串 |

### 使用示例

```python
system_prompt = SystemPromptContext()

# 注册用户自定义段
system_prompt.register_segment(PromptSegment(
    id="user_custom",
    content="用户偏好：使用简洁的语言回复，不要使用 emoji。",
    priority=10,
))

# 注册工具描述段
system_prompt.register_segment(PromptSegment(
    id="tools_desc",
    content="你可以使用以下工具：...",
    priority=50,
))

# 临时禁用某段
system_prompt.disable_segment("user_custom")
```

### 未来扩展

- 从文件加载用户自定义提示词
- 基于场景动态切换段落组合

## 3. LongTermMemoryContext — 长期记忆

**文件**: `src/core/context/modules/long_term_memory.py`

对接 `MemoryStore`（飞书文档存储），将用户画像/偏好等长期记忆注入 LLM 上下文。

### 接口

| 方法 | 说明 |
|------|------|
| `refresh()` | 从 MemoryStore 刷新缓存 |
| `format()` | 格式化为 `source='long_term_memory'` 的 ContextItem |

### 特点

- 只读注入：写入操作通过 MemoryStore / memory 工具完成
- 自动获取：`format()` 时如果内部缓存为空，会直接从 MemoryStore 读取

---

## 4. ShortTermMemoryContext — 短期记忆

**文件**: `src/core/context/modules/short_term_memory.py`

管理对话历史和 LLM 压缩，替代旧版 ContextManager 中的 `_messages` + `_summary`。

### 构造参数

| 参数 | 类型 | 说明 |
|------|------|------|
| storage | IContextStorage | 持久化存储实现 |
| compressor | ContextCompressor | 压缩策略（可选） |
| token_estimator | TokenEstimator | Token 估算器（可选） |

### 核心方法

| 方法 | 说明 |
|------|------|
| `append_message(item)` | 同时写入内存和持久化存储 |
| `needs_compression(max_tokens, threshold)` | 检查是否需要压缩 |
| `estimate_tokens()` | 估算当前 token 用量 |
| `compress(summarize_fn, keep_ratio)` | 执行 LLM 压缩 |
| `clear()` | 清空并开启新会话 |
| `format()` | 返回 [摘要(如有)] + 历史消息 |

### 压缩流程

1. 计算分割点（保留最近 30% 的消息）
2. 调整分割点（不在 tool_calls 中间截断）
3. 用 LLM 对旧消息生成摘要
4. 保存 checkpoint 到持久化存储
5. 替换内存中的消息列表

## 5. ToolContext — 工具消息序列管理

**文件**: `src/core/context/modules/tool_context.py`

管理当前对话轮次中的工具调用链（assistant tool_calls + tool responses）。

### 核心方法

| 方法 | 说明 |
|------|------|
| `add_tool_call(item)` | 添加含 tool_calls 的 assistant 消息 |
| `add_tool_response(item)` | 添加 tool 响应消息 |
| `has_pending_calls()` | 是否有未匹配的 tool_calls |
| `sanitize()` | 清理不完整的工具调用链 |
| `archive_to(short_term)` | 将当前工具链归档到短期记忆 |
| `format()` | 返回当前轮次的所有工具消息 |

### 生命周期

每轮对话中：
1. tool_loop 产生的 intermediate messages 通过 `ContextManager.append_message()` 路由到 ToolContext
2. tool_loop 结束后，Agent 调用 `ctx.archive_tool_context()` 将工具消息归档到 ShortTermMemory
3. ToolContext 被清空，准备下一轮

---

## 6. ContextManager — 统一管理器

**文件**: `src/core/context/manager.py`

协调所有上下文模块，提供面向 Agent 的统一接口。

### 公开 API

| 方法 | 说明 |
|------|------|
| `append_message(msg_dict)` | 追加消息，自动路由到合适的模块 |
| `get_context()` | 构建完整 LLM 上下文（返回 `list[dict]`） |
| `needs_compression()` | 检查是否需要压缩 |
| `compress(summarize_fn)` | 触发压缩 |
| `estimate_tokens()` | 估算当前上下文的 token 用量 |
| `clear_conversation()` | 清空对话，开启新会话 |
| `archive_tool_context()` | 归档工具上下文到短期记忆 |

### 消息路由逻辑

`append_message()` 根据消息内容自动路由：

- `role=assistant` + `tool_calls` → `ToolContext.add_tool_call()`
- `role=tool` → `ToolContext.add_tool_response()`
- 其他 → `ShortTermMemoryContext.append_message()`

### 上下文组装

`get_context()` 内部调用 `_build_context_items()` 按以下顺序组装：

1. `SystemPromptContext.format()` — 系统指令
2. `LongTermMemoryContext.format()` — 长期记忆
3. `ShortTermMemoryContext.format()` — 摘要 + 对话历史
4. `ToolContext.format()` — 当前轮工具消息

然后通过 `_to_messages()` 转换为 dict 列表，最后用 `sanitize_messages()` 清理。
