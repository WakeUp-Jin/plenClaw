# 数据结构与接口

## 新增的数据结构

### SystemPart

一段带身份标识的 system 级内容，最终合并进唯一的 system message。

**文件**: `src/core/context/types.py`

```python
@dataclass
class SystemPart:
    tag: str          # XML 标签名，如 "long_term_memory"
    description: str  # 描述文字，如 "以下是你对用户的长期记忆"
    content: str      # 实际内容文本
```

三个字段的职责：

| 字段 | 用途 | 示例 |
|------|------|------|
| `tag` | XML 标签名，包裹内容 | `"system_prompt"`, `"long_term_memory"`, `"conversation_summary"` |
| `description` | 描述文字，写入 XML 属性 | `"以下是你对用户的长期记忆，请基于这些信息个性化回复"` |
| `content` | 实际内容 | 提示词文本、记忆文本、摘要文本等 |

#### render() 方法

渲染为带 XML 标签的文本段落：

```python
def render(self) -> str:
    if self.description:
        return f'<{self.tag} description="{self.description}">\n{self.content}\n</{self.tag}>'
    return f"<{self.tag}>\n{self.content}\n</{self.tag}>"
```

有 description 时：

```xml
<long_term_memory description="以下是你对用户的长期记忆，请基于这些信息个性化回复">
用户偏好：中文回复
</long_term_memory>
```

没有 description 时（如核心提示词，本身就是指令）：

```xml
<system_prompt>
你是 PineClaw，一个AI 助手...
</system_prompt>
```

### ContextParts

模块 `format()` 的返回类型，区分内容的投递目标。

**文件**: `src/core/context/types.py`

```python
@dataclass
class ContextParts:
    system_parts: list[SystemPart] = field(default_factory=list)
    message_items: list[ContextItem] = field(default_factory=list)
```

| 字段 | 类型 | 最终去向 |
|------|------|---------|
| `system_parts` | `list[SystemPart]` | 合并为 1 条 system message |
| `message_items` | `list[ContextItem]` | 逐个 `to_message()` 放入消息列表 |

## 接口变化

### BaseContext.format()

**文件**: `src/core/context/base.py`

改动前：

```python
@abstractmethod
def format(self) -> list[ContextItem]:
    """将存储项格式化为 ContextItem 列表"""
```

改动后：

```python
@abstractmethod
def format(self) -> ContextParts:
    """子类返回 ContextParts，声明内容投递到 system 还是 messages"""
```

所有子类的 `format()` 返回值从 `list[ContextItem]` 改为 `ContextParts`。

## 各模块的 format() 实现

### SystemPromptContext

```python
def format(self) -> ContextParts:
    prompt = self.get_prompt()  # 按优先级合并所有启用的 PromptSegment
    if not prompt:
        return ContextParts()
    return ContextParts(system_parts=[
        SystemPart(tag="system_prompt", description="", content=prompt),
    ])
```

核心提示词没有 description——它本身就是指令，不需要额外解释。

### LongTermMemoryContext

def format(self) -> ContextParts:

### ShortTermMemoryContext

```python
def format(self) -> ContextParts:
    parts = ContextParts()

    if self._summary:
        parts.system_parts.append(SystemPart(
            tag="conversation_summary",
            description="以下是之前对话的压缩摘要",
            content=self._summary,
        ))

    parts.message_items.extend(self._items)
    return parts
```

这是区分最明显的地方——压缩摘要是 system 级背景信息，投递到 `system_parts`；对话消息是消息流，投递到 `message_items`。
