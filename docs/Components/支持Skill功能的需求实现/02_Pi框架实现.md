# Pi 框架实现：原生 Skill 系统集成

## 概述

Pi（`@mariozechner/pi-coding-agent`）是一个极简的终端编程助手，**原生遵循 [Agent Skills 标准](https://agentskills.io)**。
你不需要自己实现 Skill 的渐进加载和触发机制——Pi 已经内置了完整的 Skill 支持。

**Pi 的核心哲学：**
> 不内置 Sub-agents、Plan Mode、MCP——但提供 Extension API，让你用 TypeScript 自己实现这些能力。

**默认提供的 4 个工具：**`read` / `write` / `edit` / `bash`（还有 `grep`, `find`, `ls`）

**支持的模型提供商：**Anthropic、OpenAI、Google、Mistral、Groq、Bedrock 等 20+

## Pi 的 Skill 系统

### Skill 目录发现

Pi 自动从以下目录加载 Skills（按优先级）：

```
.agents/skills/          ← 项目级（当前目录及父目录向上查找）
.pi/skills/              ← 项目级（pi 专属）
~/.agents/skills/        ← 全局（agentskills.io 标准路径）
~/.pi/agent/skills/      ← 全局（pi 专属）
pi package 安装目录        ← 通过 pi install 安装的包
```

## Pi 的脚本执行链路

Pi 的 `bash` 工具会在**子进程**中执行命令：

```
Skill 告知脚本路径（在 SKILL.md 中写明）
    ↓
Pi 将 Skill 内容注入 system prompt
    ↓
LLM 决定调用 bash 工具
    ↓
Pi 的 bash 工具（子进程）执行脚本
    ↓
stdout/stderr 返回给 LLM 继续推理
```

**Pi 的特殊之处**：Pi 故意没有后台 bash（哲学：用 tmux 获得完全可见性），每次 bash 工具调用都是前台同步执行。

## Pi SDK 模式（嵌入到 Python/TypeScript 应用）

Pi 提供 SDK，可以将其嵌入到你自己的应用中：

```typescript
// TypeScript SDK
import {
  AuthStorage,
  createAgentSession,
  ModelRegistry,
  SessionManager
} from "@mariozechner/pi-coding-agent";

const { session } = await createAgentSession({
  sessionManager: SessionManager.inMemory(),
  authStorage: AuthStorage.create(),
  modelRegistry: new ModelRegistry(authStorage),
  // Skill 自动从标准目录加载，无需额外配置
});

// 发送消息（Skill 系统自动触发）
await session.prompt("帮我创建一个 FastAPI 项目");
```

**从 Python 调用 Pi（RPC 模式）**：

```python
import subprocess
import json

class PiAgent:
    """通过 Pi 的 RPC 模式在 Python 中使用 Pi Agent"""

    def __init__(self, provider: str = "anthropic", model: str = None):
        cmd = ["pi", "--mode", "rpc"]
        if provider:
            cmd += ["--provider", provider]
        if model:
            cmd += ["--model", model]

        self.process = subprocess.Popen(
            cmd,
            stdin=subprocess.PIPE,
            stdout=subprocess.PIPE,
            text=True,
            bufsize=1,
        )

    def send(self, message: str) -> str:
        """发送消息并等待响应"""
        request = json.dumps({"type": "prompt", "content": message}) + "\n"
        self.process.stdin.write(request)
        self.process.stdin.flush()

        responses = []
        for line in self.process.stdout:
            event = json.loads(line)
            if event.get("type") == "message":
                responses.append(event.get("content", ""))
            if event.get("type") == "done":
                break

        return "".join(responses)

    def close(self):
        self.process.terminate()


# 使用示例
agent = PiAgent(provider="openai", model="gpt-4o")
result = agent.send("帮我创建一个 FastAPI 项目")  # 自动触发 backend-api Skill
print(result)
agent.close()
```

## Pi Extension：自定义工具和子代理

当需要超出 Pi 默认能力时，通过 TypeScript Extension 扩展：

```typescript
// ~/.pi/agent/extensions/custom-tools.ts
export default function (pi: ExtensionAPI) {

  // 注册自定义工具
  pi.registerTool({
    name: "deploy",
    description: "Deploy the project to the server",
    inputSchema: {
      type: "object",
      properties: {
        environment: { type: "string", enum: ["staging", "production"] }
      },
      required: ["environment"]
    },
    execute: async ({ environment }) => {
      // 执行部署逻辑
      return `Deployed to ${environment} successfully`;
    }
  });

  // 监听工具调用事件（类似 Hooks）
  pi.on("tool_call", async (event, ctx) => {
    if (event.tool === "bash" && event.input.command.includes("rm -rf")) {
      // 危险命令确认
      const confirmed = await ctx.confirm("确认执行危险命令？");
      if (!confirmed) return ctx.abort();
    }
  });

  // 注册自定义命令
  pi.registerCommand("skill-list", {
    description: "列出所有已加载的 Skills",
    execute: async (ctx) => {
      // 自定义命令逻辑
      ctx.print("已加载的 Skills：...");
    }
  });
}
```

**实现子代理（Pi 本身不内置，但可以通过 Extension 自建）**：

```typescript
// extensions/sub-agent.ts
export default function (pi: ExtensionAPI) {
  pi.registerTool({

    description: "Spawn a specialized sub-agent for a specific task",

      properties: {
        task: { type: "string" },
        skill: { type: "string", description: "Skill name to load for sub-agent" }
      },
      required: ["task"]
    },
    execute: async ({ task, skill }) => {
      // 启动新的 pi 实例作为子代理
      const result = await runPiSubprocess(task, skill);
      return result;
    }
  });
}
```

## Pi Package：打包发布 Skill

将 Skill 打包为 npm package 供他人使用：

```json
// package.json
{
  "name": "@yourname/pi-backend-skills",
  "keywords": ["pi-package"],
  "pi": {
    "skills": ["./skills"],
    "extensions": ["./extensions"]
  }
}
```

```bash
# 安装他人的 Skill 包
pi install npm:@yourname/pi-backend-skills

# 列出已安装
pi list

# 更新
pi update
```

## 与自建实现的对比

| 方面 | Pi 框架 | 自建 |
|------|---------|------|
| Skill 加载 | 自动（内置） | 需自己写 ~100 行 |
| 触发检测 | 自动（内置） | 需自己写 ~20 行 |
| bash 工具 | 内置 | 需自己写 ~20 行 |
| 多模型 | 原生支持 20+ | 需适配不同 SDK |
| 自定义工具 | TypeScript Extension | Python 函数 |
| 子代理 | 需 Extension 实现 | 完全自控 |
| 语言 | TypeScript（Extension） | 纯 Python |
| 学习深度 | 浅（直接用） | 深（理解原理） |

**结论**：Pi 适合"快速集成"；自建适合"深度理解"和"完全自控"。
