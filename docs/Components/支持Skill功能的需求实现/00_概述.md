# 支持 Skill 功能的需求实现 — 总览

## 背景

Skill（技能包）是让 Agent 拥有**可插拔领域知识**的机制，遵循 [Agent Skills 标准](https://agentskills.io)。
本系列文档记录三种实现路径的设计思路与代码，最终目的是：

> 将自建实现本身抽取为一个 Skill，让任何 Agent 都能通过加载这个 Skill 来获得"支持 Skill 的能力"。

---

## 三种实现路径

| 路径 | 文档 | 语言 | 模型限制 | Skill 系统 | 适用场景 |
|------|------|------|----------|-----------|----------|
| 自建 | `01_自建实现.md` | Python | 无限制 | 完全自建 | 深度掌控、学习原理 |
| Pi 框架 | `02_Pi框架实现.md` | TypeScript | 无限制 | 原生内置 | 快速集成，无需自建 |
| Claude Agent SDK | `03_Claude_Agent_SDK实现.md` | Python/TS | 仅 Anthropic | 需自建 | 深度集成 Claude 生态 |

## 核心概念

### 什么是 Skill？

Skill 是一个目录，包含：
- `SKILL.md` — 核心内容（frontmatter 元数据 + Markdown 正文）
- `references/` — 详细参考文档（按需加载）
- `examples/` — 示例文件
- `scripts/` — 可执行脚本（由 Agent 的 bash 工具执行）

```
my-skill/
├── SKILL.md          ← 必需：元数据 + 核心指导内容
├── references/       ← 可选：详细文档（懒加载）
│   └── advanced.md
├── examples/         ← 可选：示例代码
│   └── example.py
└── scripts/          ← 可选：脚本（Agent 用 bash 工具执行）
    └── validate.sh
```

### SKILL.md 格式

```markdown
---
name: skill-identifier
description: This skill should be used when the user asks to "创建API",
  "实现路由", "配置中间件". 包含具体触发短语。
version: 0.1.0
---

# Skill 标题

## 概述
（用祈使式写作，不要用"你应该..."）

配置环境变量。
运行初始化脚本。

## 可用脚本
- 使用 bash 工具运行 `scripts/validate.sh` 进行验证
```

### 渐进式加载（Progressive Disclosure）

```
启动时     → 只加载元数据（name + description）     ~100 tokens
Skill 触发 → 加载 SKILL.md 正文                   ~2000 tokens
按需       → 加载 references/, examples/           按需 tokens
脚本       → 不加载内容，告知路径让 Agent 执行       0 tokens
```

这是控制 token 消耗的核心设计。

## Skill 与 Agent 的关系

```
┌──────────────────────────────────────────────┐
│                   Agent 系统                  │
│                                              │
│  system_prompt = [Agent 角色]                │
│               + [触发的 Skill 正文]  ←──────┐│
│                                            ││
│  ┌─────────────┐    ┌──────────────────┐  ││
│  │  Skill 系统  │───→│  触发检测 & 注入  │──┘│
│  └─────────────┘    └──────────────────┘   │
│                                            │
│  ┌─────────────────────────────────────┐   │
│  │           工具注册表                 │   │
│  │  bash / read / edit / write         │   │
│  └─────────────────────────────────────┘   │
│           ↑ Skill scripts 通过这里执行       │
└──────────────────────────────────────────────┘
```

**关键分工：**
- Skill 系统负责"把什么文本注入到 prompt"
- 工具系统负责"执行脚本"
- 两者完全解耦

---

## 脚本执行链路

Skill 里的脚本不是由 Skill 系统执行的，而是由 Agent 通过 bash 工具执行：

```
SKILL.md 写明脚本路径
    ↓
注入到 system_prompt："可用脚本：bash /path/to/validate.sh"
    ↓
LLM 决定调用 bash 工具
    ↓
bash 工具（subprocess）执行脚本
    ↓
stdout 返回给 LLM 继续推理
```

---

## 触发机制

从 `description` 字段的引号短语中提取关键词，与用户输入做匹配：

```python
# description: This skill should be used when the user asks to "创建API", "实现路由"
phrases = re.findall(r'"([^"]+)"', description)
# → ["创建API", "实现路由"]

triggered = any(p.lower() in user_input.lower() for p in phrases)
```

高级实现可以替换为向量相似度匹配（embedding）。
